## 2. 背景与目标

### 2.1 产品背景

本产品面向商业不动产运营管理场景，提供租赁全生命周期管理能力，覆盖空间规划、租户管理、合同管理、账单处理、应收与财务核算等关键业务流程。

传统模式依赖 Excel、分散系统及人工操作，存在合同录入慢、计租易出错、账务处理效率低、数据汇总不及时等问题。本产品以 AI 能力和流程自动化为核心，目标是显著提升运营效率与财务准确性。

### 2.2 产品定位

区别于传统 ERP 系统，本平台定位为“AI 原生不动产管理 SaaS 平台”，通过 OCR、NLP、计租引擎等能力，将高频重复工作自动化，支撑管理人员聚焦经营决策。

### 2.3 目标市场

- 客户类型：持有 1 栋以上商业物业的运营商或资管公司
- 管理规模：管理面积 5,000㎡ 以上
- 应用场景：写字楼、购物中心、产业园区；单一管理主体管理 1～10 个项目

### 2.4 核心业务价值（对比）

| 价值维度 | 传统方式 | 本平台目标 |
|----------|---------|------------|
| 合同录入 | 手工逐条录入，1-2 小时/份 | AI 解析，< 30 秒/份 |
| 计租计算 | 人工 Excel 核算，易出错 | 公式引擎自动计算，准确率 > 99% |
| 账单处理 | 手工生成、盖章、发送 | 全流程自动化，批量 ≥ 1000 份/次 |
| 财务对账 | 多系统数据人工核对 | 银行流水自动匹配核销 |
| 空间管理 | 纸质图纸或静态 CAD 文件 | 可视化平面图，实时状态联动 |

## 5. 功能需求

> 说明：以下每个功能模块均按照“功能描述—核心功能—验收标准”结构定义。

### 5.1 租赁单元管理

**功能描述**
实现建筑物内租赁单元数字化管理，支持图形化合并与切割。

**核心功能**
1. 基础信息管理
   - 层级管理：项目 > 楼栋 > 楼层 > 单元
   - 单元信息：编号、面积、位置、类型、状态
   - 单元属性：建筑面积、套内面积、公摊面积、层高
2. 图形化空间管理
   - 平面图导入（CAD、PDF、图片）
   - 可视化单元标注与编辑
   - 拖拽式合并：相邻单元合并、自动面积计算、生成新编号
   - 图形化切割：绘制分割线、自动面积计算、批量生成新编号
3. 状态管理
   - 状态：空置、已租、预租、装修中、维修中
   - 状态变更记录与历史追溯
   - 空置率统计与可视化

**验收标准**
- 平面图加载时间 < 3 秒
- 单次合并/切割支持单元数 ≥ 10 个
- 面积计算误差 < 0.1%

---

### 5.2 客户管理（租户管理）

**功能描述**
建立租户全量档案体系，结合 AI 自动收集与整合数据，形成 360° 客户画像。

**核心功能**
1. 租户基础信息
   - 企业信息：名称、统一社会信用代码、法人、注册资本
   - 联系信息：联系人、电话、邮箱、地址
   - 行业分类：业态、品牌定位、经营范围
2. AI 自动数据收集
   - 工商信息自动查询（企查查/天眼查 API）
   - 合同解析提取租户信息
   - 邮件往来自动归档
   - 公开信息抓取（品牌官网、公众号等）
3. 客户档案
   - 租赁历史记录
   - 履约情况（按时缴费率、违约记录）
   - 经营状况评估（营业额、客流量）
   - 信用评级（AI 评分，模型说明对用户透明）
   - 沟通记录时间线
4. 客户分类与标签
   - 自定义标签
   - A/B/C 分级管理
   - 相似业态客户智能推荐

**验收标准**
- 工商信息查询成功率 > 95%
- 合同信息提取准确率 > 90%
- 客户档案生成时间 < 5 秒

---

### 5.3 合同管理

**功能描述**
通过 AI 自动识别合同条款并解析计租规则，支持合同全生命周期管理。

**核心功能**
1. 合同录入
   - 支持 PDF/Word/图片上传
   - OCR 识别
   - 合同模板管理（可自定义）
2. AI 合同解析
   - 提取主体、租赁标的、租期、租金条款、特殊条款
   - 计租规则识别：固定、阶梯、保底+抽成、纯抽成、复合计租
   - 费用项识别：租金、物业费、水电费、空调费及附加费用
3. AI 计租引擎
   - 自动生成计租公式（示例：`月租金 = MAX(保底租金, 营业额 × 抽成比例) + 物业费 × 面积`）
   - 可视化公式编辑
   - 公式验证与测试
   - 特殊日期处理（免租期、提前解约等）
4. 生命周期与审批
   - 状态：草稿、审批中、执行中、即将到期、已到期、已终止
   - 到期提醒（30/60/90 天）
   - 续约管理、变更历史追溯
   - 自定义多级审批（串行/并行）
   - 电子签章集成（法大大/e 签宝）

**验收标准**
- 单份合同解析时间 < 30 秒
- 支持合同格式 ≥ 10 种
- 计租公式生成准确率 > 90%

### 5.4 账务管理

**功能描述**
基于合同计租规则自动生成账单，完成审核、盖章、发送与应收记录闭环。

**核心功能**
1. 自动账单生成
   - 月度定时批量生成（触发时间可配置）
   - 自动计算各费用项
   - 账单明细展示
   - 支持手工调整并记录原因
2. 账单审核
   - 预览与校验
   - 异常自动标记（金额波动超阈值、公式错误）
   - 批量审核
3. 自动盖章
   - 集成电子印章
   - PDF 自动加盖公章/财务章/合同章
   - 盖章记录可追溯
4. 自动发送
   - 邮件（模板、附件、抄送）
   - 短信通知
   - 系统内消息推送
   - 发送状态追踪
5. 应收记录与查询
   - 自动生成应收账款
   - 账龄分析
   - 逾期标记与分级催收提醒
   - 多维查询与导出（Excel/PDF）

**验收标准**
- 账单生成准确率 > 99%
- 单次批量处理账单数 ≥ 1,000 份
- 邮件发送成功率 > 98%
- 盖章处理时间 < 2 秒/份

### 5.5 财务管理

**功能描述**
提供应收管理、收入核算、回款结算与财务报表能力。

**核心功能**
1. 应收管理
   - 应收总览与明细筛选
   - 逾期管理与催收记录
2. 收入核算
   - 权责发生制确认
   - 预收款与跨期分摊
   - 多维收入统计与趋势分析
3. 回款结算
   - 收款登记（手工/导入/自动匹配）
   - 自动核销、部分核销、预收核销
   - 结算单与对账单生成
4. 财务报表
   - 应收账款、收入明细、回款统计、欠费分析
   - 自定义报表（字段、分组、排序）
5. 银行对账
   - 银行流水导入（CSV/Excel）
   - 智能匹配、未匹配项人工确认
   - 差异处理记录

**验收标准**
- 收款核销准确率 > 99.5%
- 报表生成时间 < 5 秒
- 支持并发用户数 ≥ 100
- 数据实时性延迟 < 1 分钟

### 5.6 资产地图

**功能描述**
以地图为统一入口，提供城市—楼栋—楼层—单元多层级可视化管理，支持状态联动、历史版本回溯及变更台账审计。

**核心功能**
1. 三级下钻视图
   - 第一级：地理地图视图（资产分布、聚合、筛选）
   - 第二级：建筑楼层结构视图（2.5D 楼层状态与汇总）
   - 第三级：楼层平面图视图（单元状态、浮窗详情、缩放平移）
2. 版本管理与历史回溯
   - 单元切割/合并自动生成不可篡改快照
   - 版本号规则：`楼层编号-YYYYMMDD-序号`
   - 时间轴浏览最新与历史版本
   - 双版本分屏对比与差异高亮
3. 变更记录台账
   - 记录类型、时间、操作人、前后单元、面积核对、关联合同、备注、附件
   - 支持平面图前后对比展示
   - 支持楼层/楼栋/项目维度导出（Excel/PDF）
4. 数据联动
   - 与租赁单元、合同、客户、账务模块实时联动

**验收标准**
- 地图标记加载时间（≤500 楼盘）< 3 秒
- 建筑楼层视图渲染时间 < 1 秒
- 楼层平面图加载时间 < 2 秒
- 版本切换响应时间 < 500ms
- 全项目变更记录查询 < 3 秒
- 平面图单元状态与合同数据一致性 = 100%
- 历史版本面积总量与最新版本误差 = 0

### 5.7 权限管理

**功能描述**
提供基于角色的访问控制（RBAC）与多租户隔离能力，确保租户间数据边界与权限可控。

**核心功能**
1. 角色管理
   - 预置角色：超级管理员、运营经理、招商经理、财务人员、只读用户
   - 自定义角色与权限矩阵
2. 功能权限
   - 菜单级权限（可见/不可见）
   - 操作级权限（增删改查、导出、审批等）
3. 数据权限
   - 按项目/楼栋限制访问范围
   - 支持全部/指定项目/本人项目
4. 用户管理
   - 用户创建、修改、禁用
   - 多项目成员管理
   - 账号密码+短信验证码双因素认证
   - 登录日志与异地登录告警
5. 多租户隔离
   - `tenant_id` 强制过滤
   - 租户独立品牌配置（Logo、系统名、发件人等）

**验收标准**
- 权限配置生效时间 < 5 秒
- 权限矩阵覆盖所有功能模块
- 跨租户数据隔离安全测试零漏洞
- 异地登录告警响应时间 < 1 分钟
- 增删改操作日志完整率 = 100%

#### 5.10.2 租户存储空间管理（Space）

**概念定义**

租户存储空间（Space）是平台数据隔离的核心单元，位于数据库实例和租户之间，形成三层架构：

```
数据库实例（DB Instance）
  └── 租户存储空间（Space = PostgreSQL Schema）
        └── 租户（Tenant，行级隔离 tenant_id）
        └── 租户（Tenant，行级隔离 tenant_id）  ← 共享租户存储空间
  └── 租户存储空间（Space = PostgreSQL Schema）
        └── 租户（Tenant，独占）               ← 专属租户存储空间
```

- 每个租户存储空间对应数据库实例中的一个独立 Schema
- 多个租户可共享同一租户存储空间（行级隔离，通过 `tenant_id` 过滤）
- 也可为单个租户分配专属租户存储空间（Schema 级隔离）
- 配置租户时，平台管理员可选择将该租户接入哪个租户存储空间

---

##### 5.10.2.1 租户存储空间基础信息

|------|------|------|
| space_id | 租户存储空间唯一标识 | `space_cn_shared_01` |
| 租户存储空间名称 | 描述性名称，供平台内部识别 | 华东共享池-01 |
| 租户存储空间类型 | `shared`（共享）/ `dedicated`（专属） | shared |
| 数据库实例 | 所在 DB 实例标识（主机/连接串引用） | `db-instance-cn-east-01` |
| Schema 名称 | 数据库中的实际 Schema 名 | `space_cn_shared_01` |
| 地域 | 数据所在地域，用于合规与性能 | 华东（上海） |

| 存储用量 | 当前已用存储（GB） | 38.4 GB |

##### 5.10.2.2 租户存储空间类型对比

| 维度 | 共享租户存储空间（Shared） | 专属租户存储空间（Dedicated） |
|------|-------------------|----------------------|
| 隔离级别 | 行级（tenant_id） | Schema 级 |
| 适用套餐 | 试用版、标准版、专业版 | 旗舰版（Enterprise）|
| 数据物理隔离 | 否（共用表） | 是（独立表结构） |
| 性能保障 | 共享资源，无独立 QoS | 可独立配置连接池、资源限额 |
| 成本 | 低 | 高 |
| 迁移难度 | 可迁移至专属（需数据搬迁） | 可合并回共享（极少场景） |
| 最大租户数 | 由租户存储空间容量上限控制 | 1 |

├─────────────────────────────────────────────────────────┤

└─────────────────────────────────────────────────────────┘

**选择规则**

**自动推荐逻辑**
系统按以下优先级自动推荐租户存储空间，管理员可手动覆盖：
1. 同地域、类型匹配、当前占用率 < 70% 的租户存储空间
2. 按当前租户数升序排列（优先填满租户存储空间，减少碎片）

##### 5.10.2.4 数据访问路由

运行时，系统通过以下链路将请求路由至正确的 Schema：

```
HTTP 请求（携带 tenant_id / JWT）
  ↓
API 网关 / 中间件：解析 tenant_id
  ↓
Space Router（查询平台元数据库）
  tenant_id → space_id → db_instance + schema_name
  ↓
业务查询（动态设置 search_path = {schema_name}）
  ↓
所有 SQL 自动附加 WHERE tenant_id = :tid（共享租户存储空间强制过滤）
```

**元数据缓存**：`tenant_id → space` 的路由映射缓存于 Redis，TTL = 5 分钟，租户存储空间变更时主动失效。

##### 5.10.2.5 租户存储空间迁移（Tenant Space Migration）

当租户需要从共享租户存储空间迁移到专属租户存储空间时（如升级旗舰版），执行以下流程：

```
1. 平台管理员创建迁移任务
   ├─ 源租户存储空间：space_cn_shared_01
   ├─ 目标租户存储空间：space_cn_ded_xinghe（新建或已有空闲专属租户存储空间）
   └─ 预计停服窗口：系统自动评估（一般 < 30 分钟）

2. 迁移预检（自动）
   ├─ 目标租户存储空间 Schema 结构版本一致性检查
   ├─ 源数据量评估（预计迁移时长）
   └─ 通知租户管理员：XX 分钟后进入维护窗口

3. 维护窗口执行
   ├─ 租户状态置为"迁移中"（只读，不可登录操作）
   ├─ 数据全量导出（SELECT WHERE tenant_id = :tid）
   ├─ 写入目标 Schema
   ├─ 数据校验（行数、关键字段 Checksum）
   └─ 路由表切换：tenant_id → 新 space_id（Redis 立即失效旧缓存）

4. 迁移完成
   ├─ 租户状态恢复正常
   ├─ 旧租户存储空间中该租户数据保留 7 天后清理（归档备份）
   └─ 通知租户管理员迁移完成
```

**迁移约束**
- 迁移期间租户数据只读，不可跨地域迁移
- 迁移失败自动回滚（路由不切换，通知管理员）
- 同一租户同一时间只能有一个迁移任务

操作流程：
1. 客户在系统设置中填写自定义域名（如 `rms.xinghe.com`）
2. 系统生成 CNAME 验证值，展示给客户
3. 客户在自己的 DNS 服务商添加 CNAME 记录：`rms.xinghe.com → cname.mdos.com`
4. 平台每 5 分钟轮询验证 CNAME 是否生效
5. 验证通过后：自动签发 SSL 证书，域名状态变为"已生效"
6. 两个域名（子域名 + 自定义域名）同时有效

| 状态 | 说明 |
|------|------|
| 待验证 | 已填写域名，等待 CNAME 生效 |
| 验证中 | 检测到 CNAME，SSL 签发中 |
| 已生效 | 域名可正常访问 |
| 验证失败 | CNAME 不正确或超时（24h） |
| 已停用 | 订阅到期后自定义域名停止解析 |

#### 5.10.6 平台管理后台功能

> 该后台仅限 mdos 平台运营人员访问，独立于各租户系统。

5. 套餐管理（新增/修改/下架套餐）

**验收标准**

| 验收项 | 标准 |
|--------|------|
| slug 唯一性校验 | 重复 slug 实时提示，不可提交 |
| 子域名生效时间 | 租户创建后 < 60 秒可访问 |
| CNAME 验证轮询 | 每 5 分钟检测一次，验证通过后 10 分钟内 SSL 签发完成 |
| 订单激活时效 | 支付成功后 < 30 秒租户状态变为"正常使用" |
| 到期提醒准时率 | 30/15/7 天提醒发送误差 < 1 小时 |
| 宽限期数据保护 | 宽限期内数据完整，不可删除、不可写入 |
| 跨租户域名隔离 | 同一自定义域名不可绑定两个租户 |
| 平台后台访问控制 | 非平台运营账号访问返回 403 |
| 租户存储空间路由缓存命中率 | tenant_id → space 路由缓存命中率 ≥ 99%，路由延迟 < 5ms |
| 跨租户存储空间数据隔离 | 安全测试：A 租户存储空间租户查询不可访问 B 租户存储空间任何数据，零漏洞 |
| 租户存储空间创建自动化 | 创建租户存储空间后 `CREATE SCHEMA` 执行完成时间 < 10 秒 |
| 租户迁移数据完整性 | 迁移后数据行数与 Checksum 与源租户存储空间一致，误差 = 0 |
| 迁移停服窗口 | 10 万行数据以内迁移停服时间 < 30 分钟 |
| 已满租户存储空间不可选 | 创建租户时 `full` 状态租户存储空间不显示在可选列表中 |

## 6. 非功能需求

### 6.1 性能要求

| 场景 | 指标 |
|------|------|
| 常规页面加载 | < 2 秒 |
| 复杂查询（报表） | < 5 秒 |
| 批量账单生成（1000 份） | < 60 秒 |
| 并发用户数 | ≥ 500 |
| 大文件上传上限 | ≤ 50MB |

### 6.2 安全与合规要求

- 传输加密：HTTPS / TLS 1.2+
- 存储加密：敏感数据 AES-256
- 权限体系：角色权限 + 数据权限（见 5.7）
- 审计日志：全量记录，保留 1 年
- 备份策略：每日增量 + 每周全量
- 合规要求：符合《数据安全法》《个人信息保护法》
- 认证目标：正式版本前完成等保二级认证

### 6.3 可用性与容灾要求

- 系统可用性 ≥ 99.5%（年停机 ≤ 43.8 小时）
- RTO < 4 小时
- RPO < 1 小时
- 备份保留 ≥ 90 天

### 6.4 兼容性要求

- 浏览器：Chrome、Edge、Safari、Firefox（最新两个版本）
- 移动端：iOS 14+、Android 10+（资产地图完整能力适配平板及以上）
- 最低分辨率：1920×1080

### 6.5 扩展性要求

- 微服务架构，支持模块化独立部署
- 开放 REST API（含文档与 SDK）
- 支持第三方集成（财务系统、OA、ERP）
- 支持 Webhook 事件推送

### 6.6 国际化与本地化要求

**支持语言**
- **第一阶段（MVP）**：简体中文、英文（英美）
- **第二阶段**：繁体中文（香港/台湾）、日文、韩文、越南文、泰文
- **第三阶段**：欧语（法文、德文、西班牙文），中东/东南亚语言

**前端国际化**
- 所有用户界面文本、标签、提示、错误消息均支持多语言切换
- 语言切换响应时间 ≤ 1 秒，无页面刷新
- 用户语言偏好持久化存储（可在个人设置中修改）
- 支持浏览器 Accept-Language 自动检测并推荐语言

**后端 API 国际化**
- 所有 API 响应消息（code、message 字段）按 Accept-Language header 返回
- 业务异常提示消息国际化，返回消息键供客户端国际化处理
- 枚举值翻译（如订单状态、用户角色等）随 API 返回

**数据与格式国际化**
- 日期时间格式配置：支持 12h/24h 制、ISO 8601 等标准格式
- 货币格式：支持 CNY、USD、EUR 等多币种符号与格式规范
- 数字分隔：支持逗号或空格分隔（如 1,000 vs 1 000）
- 地区特定设置：时区、公假日历（基于国家/地区）

**多语言字段存储**
- 支持 JSONB 多语言存储：`field_i18n: { zh: "...", en: "...", ja: "..." }`
- 默认语言优先，缺失时降级到英文，都缺失时返回原始数据

**翻译管理**
- 提供平台内翻译管理后台（供运营人员维护翻译）
- 支持翻译导入/导出（Excel、JSON 等格式）
- 支持翻译版本管理与修订历史追踪

**验收标准**
- 支持的语言在登录页、主菜单、个人设置中可见
- 语言切换后所有界面元素立即更新，无刷新
- 日期、货币、数字格式符合各地区习惯
- API 响应消息 100% 国际化覆盖
- 缺失翻译时能优雅降级（显示默认语言或键名）
- 支持 ≥ 5 种语言无性能相关影响（加载时间 < 100ms)
- 不同语言用户并发访问隔离正确
